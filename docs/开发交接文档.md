# 源师云课 · 智慧云课堂平台 - 开发交接文档

## 1. 项目概述

本项目为高校智慧云课堂平台前端（品牌名：**源师云课**），当前已完成：

- 官网展示页（参考 `safelogin-homepage.html` 风格）
- 学生端核心页面（AI数字人教授课堂 P0、学情分析）
- 教师端核心页面（AI智能出题）
- 管理端核心页面（全局数据看板）

当前阶段定位为**前端原型与交互验证**，数据层已采用 mock 接口，便于后续接入真实后端。

## 4. 路由与页面清单

当前路由统一挂在 `RootLayout` 下，通过分组路由组织：

- 官网

## 7. 后续重点：登录与角色分流改造

## 7.1 改造目标

- 游客可访问官网
- 游客点击服务交互跳转登录页
- 登录后根据角色自动分流：
  - 学生 -> 学生端
  - 教师 -> 教师端
  - 管理员 -> 管理端
- 角色越权访问时拦截（403 或重定向）

## 7.2 建议技术方案（轻量）

1. 新增 `AuthContext`（`isAuthenticated`、`role`、`userInfo`）
2. 新增登录页 `/login`
3. 新增守卫组件：
   - `AuthGuard`：未登录拦截
   - `RoleGuard`：角色不匹配拦截
4. 按角色拆 Layout：
   - `PublicLayout`（官网/登录）
   - `StudentLayout`
   - `TeacherLayout`
   - `AdminLayout`
5. 官网按钮统一封装鉴权跳转逻辑（未登录先去登录）

## 7.3 建议落地顺序

1. 先加登录页 + 假登录（mock）
2. 再加路由守卫
3. 最后拆导航与角色菜单

这样能降低改动风险，便于逐步验证。
